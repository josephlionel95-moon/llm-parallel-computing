# Chapter 09 references — DDP internals

Annotated bibliography. Order = suggested reading order.

## Core

1. **Li, S. et al. — "PyTorch Distributed: Experiences on Accelerating Data
   Parallel Training" (VLDB 2020).**
   THE paper for this chapter: hooks, buckets, reverse-order heuristic,
   overlap measurements, and the discussion of gradient_as_bucket_view and
   unused parameters. Read it *after* building ManualDDP — it will read like
   a design review of your own code.

2. **PyTorch documentation — "DistributedDataParallel" API doc + "DDP
   Internal Design" note (pytorch.org/docs → notes/ddp).**
   The flag-by-flag truth: find_unused_parameters' graph traversal,
   static_graph's promises, broadcast_buffers, no_sync, join(). The internal
   design note has the Reducer state machine our chapter paraphrases.

3. **PyTorch source — `torch/nn/parallel/distributed.py` and
   `torch/csrc/distributed/c10d/reducer.cpp`.**
   Read `Reducer::mark_variable_ready` and `Reducer::finalize_backward`
   against `ManualDDP._hook` and `ManualDDP.finalize`. The correspondence is
   nearly line-for-line; the differences (bucket rebuilding, views, comm
   hooks) are the chapter's "production armor" list.

## Overlap and communication optimization

4. **PyTorch docs — "DDP Communication Hooks" (ddp_comm_hooks).**
   The sanctioned way to replace the bucket all_reduce: fp16/bf16 compression
   hooks, PowerSGD hook. Exercise 6 hand-rolls the same idea in ManualDDP.

5. **Vogels, T., Karimireddy, S.P., Jaggi, M. — "PowerSGD: Practical Low-Rank
   Gradient Compression for Distributed Optimization" (NeurIPS 2019).**
   What to do when comm > backward and overlap is exhausted: low-rank
   gradient approximation with error feedback, deployed in real large runs.

6. **Sergeev, A., Del Balso, M. — "Horovod" (2018), §"tensor fusion".**
   Independent convergent design of bucketing + overlapped ring all-reduce.
   Comparing Horovod's fusion buffer heuristics with DDP's bucket_cap_mb is a
   good exercise in recognizing the same α–β forces.

## Where this machinery goes next

7. **Rajbhandari, S. et al. — "ZeRO" (SC 2020).**
   Stage 2 = this chapter's bucketed gradient reduction, but reduce_scatter
   instead of all_reduce, each rank retaining only its shard. Read §3–5 as a
   preview; ch15 implements it.

8. **Zhao, Y. et al. — "PyTorch FSDP: Experiences on Scaling Fully Sharded
   Data Parallel" (VLDB 2023).**
   FSDP = DDP's hook/overlap machinery rebuilt around
   all_gather(params)/reduce_scatter(grads) with prefetching. Every concept
   in it (flat params ≈ buckets, backward prefetch ≈ reverse-order fills) is
   a renamed friend from this chapter. ch16's core reading.

9. **Ansel, J. et al. — "PyTorch 2: Faster Machine Learning Through Dynamic
   Python Bytecode Transformation and Graph Compilation" (ASPLOS 2024),
   compiled-autograd sections.**
   Where bucket heuristics head when the framework can see the whole backward
   graph ahead of time (ch27 context).
