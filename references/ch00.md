# References — Chapter 00: Python memory, processes & the GIL

## Primary documentation

- **CPython docs — Data model** (docs.python.org/3/reference/datamodel.html) — the authoritative definition of objects, identity, and names; settles every "is this a copy?" argument.
- **CPython docs — `copy` module** (docs.python.org/3/library/copy.html) — precise shallow-vs-deep semantics, including how `__copy__`/`__deepcopy__` hooks work.
- **CPython docs — `pickle` module** (docs.python.org/3/library/pickle.html) — read "What can be pickled and unpickled?" and the security warning; explains why lambdas and open handles fail.
- **CPython docs — `multiprocessing`** (docs.python.org/3/library/multiprocessing.html) — start methods (fork/spawn/forkserver), programming guidelines; the "safe importing of main module" section explains the `if __name__ == "__main__"` requirement.
- **PyTorch docs — Multiprocessing best practices** (pytorch.org/docs/stable/notes/multiprocessing.html) — the single most load-bearing page for this chapter: CUDA-and-fork, shared-memory strategies, queue caveats for CUDA tensors.
- **PyTorch docs — `torch.multiprocessing`** (pytorch.org/docs/stable/multiprocessing.html) — documents the custom tensor reducers that ship shm/CUDA-IPC handles instead of bytes.
- **PyTorch docs — `torch.Tensor.data_ptr`, Tensor views** (pytorch.org/docs/stable/tensor_view.html) — the official list of view-creating ops; memorize it.
- **PyTorch docs — DataLoader** (pytorch.org/docs/stable/data.html) — the "Multi-process data loading" section describes exactly what crosses worker boundaries and when.

## Deeper background

- **David Beazley, "Understanding the Python GIL" (PyCon 2010, dabeaz.com/GIL)** — the classic talk; shows *why* threads can be slower than one thread on multicore machines (GIL thrashing).
- **PEP 703 — Making the GIL optional in CPython** (peps.python.org/pep-0703) — best available explanation of what the GIL actually protects (refcounts, allocator state) and the cost of removing it.
- **PEP 3148 / `concurrent.futures` docs** — the executor abstraction used in our benchmarks; the clean way to switch threads ↔ processes and observe the GIL's effect.
- **NVIDIA CUDA C Programming Guide — "CUDA context" and driver API sections** (docs.nvidia.com/cuda) — background for why a forked child cannot inherit a live CUDA context.
- **`/dev/shm` and POSIX `shm_overview(7)` man page** — what `share_memory_()` actually allocates on Linux; useful when Kaggle/Colab shm limits bite (DataLoader "bus error" failures).

## For the curious

- **CPython source: `Objects/object.c`, `Include/object.h`** — see `ob_refcnt` with your own eyes; ten minutes here demystifies `id()` and the GIL's purpose.
- **Antoine Pitrou, PEP 574 — pickle protocol 5 out-of-band buffers** — how modern pickle avoids copying large buffers; the design that makes tensor serialization cheap.
