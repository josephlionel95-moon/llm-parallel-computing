# References — Chapter 02: Autograd, hooks & checkpointing

## Core (read these)

- **PyTorch docs — Autograd mechanics** (pytorch.org/docs/stable/notes/autograd.html) — leaves vs non-leaves, accumulation, in-place rules, version counters; the official contract this chapter explains.
- **PyTorch docs — `torch.autograd.Function` / "Extending PyTorch"** (pytorch.org/docs/stable/notes/extending.html) — the rules for custom forward/backward (`save_for_backward`, `mark_dirty`, one-grad-per-input) with rationale.
- **PyTorch docs — `torch.utils.checkpoint`** (pytorch.org/docs/stable/checkpoint.html) — reentrant vs non-reentrant semantics, RNG preservation; explains every checkpointing gotcha we listed.
- **Chen, Xu, Zhang & Guestrin (2016), "Training Deep Nets with Sublinear Memory Cost" (arXiv:1604.06174)** — the √L recomputation schedule; short and worth deriving alongside exercise 6.
- **Li et al. (2020), "PyTorch Distributed: Experiences on Accelerating Data Parallel Training" (VLDB, arXiv:2006.15704)** — §3 documents DDP's per-parameter autograd hooks and bucketing — the payoff of this chapter's hook material.

## Mechanics & internals

- **Paszke et al. (2019), "PyTorch: An Imperative Style, High-Performance Deep Learning Library" (NeurIPS)** — design rationale for define-by-run autograd and the C++ engine.
- **Paszke et al. (2017), "Automatic differentiation in PyTorch" (NIPS-W)** — the original short autograd note; the grad_fn/next_functions structure in 4 pages.
- **PyTorch docs — Autograd hooks / `register_post_accumulate_grad_hook`** (pytorch.org/docs/stable/generated/torch.Tensor.register_post_accumulate_grad_hook.html) — the public API closest to what DDP uses internally.
- **PyTorch docs — `torch.autograd.graph.saved_tensors_hooks` & `save_on_cpu`** — the interception point for saved activations; our CPU-safe activation scale and the basis of activation offloading.
- **PyTorch docs — CUDA memory management** (pytorch.org/docs/stable/notes/cuda.html#memory-management) — allocated vs reserved, `PYTORCH_CUDA_ALLOC_CONF`, `expandable_segments`; the allocator section's primary source.
- **PyTorch blog — "Understanding GPU Memory" series (2023)** — memory snapshot tooling and fragmentation case studies with pictures; a preview of ch24.

## Background & classics

- **Griewank & Walther (2000), "Algorithm 799: revolve"** — optimal checkpointing schedules from the AD literature; where the ML papers' ideas come from.
- **Baydin et al. (2018), "Automatic Differentiation in Machine Learning: a Survey" (JMLR)** — situates reverse-mode AD formally; read §3 to see backward() as vector–Jacobian products.
- **torch.compile / AOTAutograd docs** — how graph capture coexists with the eager autograd model described here; useful once you ask "does any of this survive compilation?" (it does).
